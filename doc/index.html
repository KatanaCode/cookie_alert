CookieAlert (beta)

CookieAlert is designed to notify visitors to your website that you use Cookies. It:

  • Will work 'straight out the box' without any additional configuration

  • Is fully customisable, including views, javascript response and number of displays.

  • Aims to be a simple way to address the UK Law requiring websites to notify their visitors that they use Cookies.

Installation

Add this to your Gemfile

gem "cookie-alert"

then type bundle install to install the gem.

Next:

  • Add include CookieAlert to your app/helpers/application_helper.rb file.

  • Add <%= display_cookie_alert %> to the body section in your app/layouts/application.rb file

  • Add <%= stylesheet_link_tag "cookie_alert" %> to the head section in your app/layouts/application.rb file

  • Add mount CookieAlert::Engine => "/cookie-alert" to your config/routes.rb file

Restart your aplication and your good to go.

How It Works

When a visitor first comes to your site CookieAlert will set a Cookie that keeps track of how many pages the visitor has viewed, and display a primary alert notifying the visitor that your site uses Cookies. By default this primary alert is a large banner fixed at the foot of the screen.

After the primary alert has been displayed a number of times (5 by default) a secondary alert is rendered instead. By default this is a smaller box fixed to the bottom-left of the screen which transforms back to the the large banner when moused-over.

The visitor can click a link to 'hide the banner', thereby accepting the warning, and the alerts will ne longer be displayed for the duration of the visitors session.

Dependencies

The default alerts uses JQuery to allow a server-side response and to add effects. JQuery is not a required dependency within the gem however, as you may cofigure the JavaScript reponse template to use any other JavaScript framework you choose.

Configuration

CookieAlert can be configured to:

  • Display a primary alert message until the visitor clicks the 'accept' link.

  • Display a primary alert message then, after a configurable number of views, change to a secondary alert message until the visitor clicks the 'accept' link.

  • Display a primary alert message then, after a configurable number of views, stop showing any alert messages.

You can also:

  • Copy the primary alert, secondary alert and JavaScript response templates to your application and completely change them to suit your own requirements.

  • Copy the CSS file to your applications and change it at will.

  • Install a configuration file that will let you choose exactly how CookieAlert should work.

  • You can even use your own existing templates for the primary and secondary alerts.

Installing The Configuration Files

To install the configuration options run:

$ rails g cookie_alert:install


 ===========================================
     Starting CookieAlert installation
 ===========================================

 Copy default configuration file to '/config/initializers/cookie_alert.rb' ? [y/n]

Enter 'y' if you want to install the cofiguration file in your config\initializers folder. This will let you cofigure the number of alerts to display, whether to use a secondary alert and whether the visitor must accept the cookie alert or not.

$ rails g cookie_alert:install


 ===========================================
     Starting CookieAlert installation
 ===========================================

 Copy default configuration file to '/config/initializers/cookie_alert.rb' ? [y/n]

 Copy View templates to '/app/views/cookie_alert/cookies/*' ? [y/n]

Enter 'y' if you want to install the alert templates to an app/views/cookie_alert/cookies folder. This will let you customise the look, position and content of the primary and secondary alerts. It will also let you customise how the javascript response when the visitors 'accepts' the warning.

Enter 'y' if you want to install the CSS file in your app/assets/stylesheets folder. This will let you configure the design elements of both primary and secondary alerts.

IMPORTANT

IF you are using this for a UK Website you should also include within the alert, a link to a Cookie Policy page where your visitors can view a description of the cookies used by your site.

Note that CookieAlert is a Gem to allow you to display alerts - the content and wording of the alert is entirely up to yourself. KatanaCode do not warant in any way that the default message, operation or usage of this Gem will make you compliant with the UK Cookie Law. It is up to you to ensure that!

All we can say is that we use it ourselves :-)
